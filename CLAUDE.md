## Beads Issue Tracker

This project uses **bd (beads)** for issue tracking. Run `bd prime` to see full workflow context and commands.

### Quick Reference

```bash
bd ready              # Find available work
bd show <id>          # View issue details
bd update <id> --claim  # Claim work
bd close <id>         # Complete work
```

### Rules

- Use `bd` for ALL task tracking — do NOT use TodoWrite, TaskCreate, or markdown TODO lists
- Run `bd prime` for detailed command reference and session close protocol
- Use `bd remember` for persistent knowledge — do NOT use MEMORY.md files

**Architecture in one line:** issues live in a local Dolt DB; sync uses `refs/dolt/data` on your git remote; `.beads/issues.jsonl` is a passive export. See https://github.com/gastownhall/beads/blob/main/docs/SYNC_CONCEPTS.md for details and anti-patterns.

## Session Completion

**When ending a work session**, you MUST complete ALL steps below. Work is NOT complete until `git push` succeeds.

**MANDATORY WORKFLOW:**

1. **File issues for remaining work** - Create issues for anything that needs follow-up
2. **Run quality gates** (if code changed) - Tests, linters, builds
3. **Update issue status** - Close finished work, update in-progress items
4. **PUSH TO REMOTE** - This is MANDATORY:
   ```bash
   git pull --rebase
   git push
   git status  # MUST show "up to date with origin"
   ```
5. **Clean up** - Clear stashes, prune remote branches
6. **Verify** - All changes committed AND pushed
7. **Hand off** - Provide context for next session

**CRITICAL RULES:**
- Work is NOT complete until `git push` succeeds
- NEVER stop before pushing - that leaves work stranded locally
- NEVER say "ready to push when you are" - YOU must push
- If push fails, resolve and retry until it succeeds

## What this is

**AronnaxIDE** is a native **macOS (SwiftUI)** app plus an **iOS companion**
(`AronnaxIOS`). Both are a control surface for a headless Mac mini named **kepler**:
Claude Code and Codex run *on kepler* inside **tmux**, and the app is the window onto
them. It uses subscription CLIs only (Claude Max via Claude Code, ChatGPT via Codex) —
never an API.

- Repo home (canonical): **kepler** → `/Users/kepler/Documents/AI_OS/Projects/AronnaxIDE`
- Public repo: https://github.com/kameronpence/AronnaxIDE
- macOS target `AronnaxIDE` (sources in `AronnaxIDE/`), iOS target `AronnaxIOS`
  (sources in `AronnaxIOS/`).

## Build & Test  ⚠️ READ THIS FIRST

**The source lives here on kepler, but AronnaxIDE is a Mac GUI app that MUST be built
and run ON THE MACBOOK.** Do **not** try to build or run it on kepler — running it here
launches the app *on kepler*, which is useless (the app's whole job is to run on the Mac
and drive kepler). Building on kepler is only ever a compile check, never how it ships.

**The loop to make a change (this is the normal workflow):**
1. Edit here on kepler (that's what Claude Code CLI in this dir is for), then
   `git commit` + `git push`.
2. On the **MacBook**: `git pull` → `xcodegen generate` → build/run in **Xcode-beta**
   (⌘R), or build a Release and swap `/Applications/AronnaxIDE.app`.

**Key facts:**
- The `.xcodeproj` is **gitignored** and generated by **xcodegen** from `project.yml`.
  After any checkout/branch switch: `xcodegen generate` (with no `.xcodeproj`, Xcode's
  ⌘R just error-beeps).
- Command-line build (on a Mac):
  ```bash
  DEVELOPER_DIR=/Applications/Xcode-beta.app/Contents/Developer xcodebuild \
    -project AronnaxIDE.xcodeproj -scheme AronnaxIDE -destination 'platform=macOS' build
  ```
- **SwiftTerm needs the Metal toolchain** — first build on a new Xcode fails with
  `cannot execute tool 'metal'`; fix once with
  `DEVELOPER_DIR=…/Xcode-beta.app/Contents/Developer xcodebuild -downloadComponent MetalToolchain`.
- `AronnaxIOS/Secrets.swift` is **gitignored** (kepler host/user + the app's SSH key) and
  required to compile the iOS target. It exists on kepler + the Mac; it's never committed.
- **Verifier:** run Codex on changes before finishing — `codex review --uncommitted`
  (see `AGENTS.md` / `docs/WORKFLOW.md`). Codex is the required reviewer.

## Architecture Overview

- **macOS app** (`AronnaxIDE/`): SwiftUI, single window. Sidebar (host + project switcher)
  and panes: Terminal · Coding (drives the Claude/Codex CLIs in tmux) · Browser · Vault ·
  Beads · Logs · Git/Deploy. Transport = shelling out to **`/usr/bin/ssh`** with
  ControlMaster multiplexing + ProxyJump (`SSHManager`). Projects are discovered by
  scanning `<agentWorkdir>/Projects` on the active host (`ProjectService`).
- **iOS app** (`AronnaxIOS/`): Terminal + a Claude/Codex switcher, reaching kepler over
  **Tailscale** via **Citadel** (pure-Swift SSH — iOS can't shell out). Shares the same
  tmux agent sessions as the macOS app (session name = agent + FNV-1a hash of the workdir),
  so Mac and phone attach to one live agent.
- **Persistence model:** every remote process runs in a named tmux session; the app only
  ever *attaches*, so detach/reattach and sleep/wake are lossless.

## Conventions & Patterns

- Write Swift that matches the surrounding file (naming, comment density, SwiftUI idiom).
- **Beads for all task tracking** (see above) — not markdown TODOs.
- Conventional commit messages (`feat:`, `fix:`, `refactor:`…).
- End a session by pushing (see Session Completion). The MacBook then pulls to build.
- Editing AronnaxIDE *with* AronnaxIDE is the intended workflow — see
  `docs/EDIT-IN-APP.md`.
